# 🎯 ОТВЕТ НА ВАШ ВОПРОС: Как система использует LLM?

## КРАТКИЙ ОТВЕТ

**Мы НЕ передаём файл целиком в LLM.**

Система использует **гибридный трёхуровневый подход**:

```
┌─────────────────────────────────────────────────────────┐
│  УРОВЕНЬ 1: Словарь синонимов (быстро, без LLM)         │
│  → 80-85% колонок обрабатываются здесь                  │
│  → Confidence: 0.95-0.98                                │
└─────────────────────────────────────────────────────────┘
                          ↓ (если confidence < 0.9)
┌─────────────────────────────────────────────────────────┐
│  УРОВЕНЬ 2: Эвристики (быстро, без LLM)                 │
│  → Ещё 10-15% колонок                                   │
│  → Confidence: 0.75-0.90                                │
└─────────────────────────────────────────────────────────┘
                          ↓ (если confidence < 0.6)
┌─────────────────────────────────────────────────────────┐
│  УРОВЕНЬ 3: LLM (Ollama/OpenRouter)                     │
│  → ТОЛЬКО сложные случаи (~5-10% колонок)               │
│  → Передаётся ТЕКСТОВЫЙ контекст, а не файл            │
│  → Confidence: 0.5-0.8                                  │
└─────────────────────────────────────────────────────────┘
```

---

## ЧТО ПЕРЕДАЁТСЯ В LLM?

### ✅ Передаётся:
- Название колонки (текст): `"СуммаРеализации"`
- Первые 5-10 значений (текст): `["100000", "50000", "75000"]`
- Метаданные о merged cells (текст)
- Системный промпт (жёстко задан в коде)

**Объём:** ~500-1000 токенов на запрос

### ❌ НЕ передаётся:
- Весь файл целиком
- Бинарные данные
- Все строки данных
- Конфиденциальная информация клиентов

## АРХИТЕКТУРА ПО ШАГАМ

### Шаг 1: Извлечение образца (БЕЗ LLM)
```python
# ExcelParser читает файл через openpyxl/pandas
sample = {
    "sheet_name": "Отчет по продажам",
    "merged_cells": [{"top_left": "R1C1", "value": "Январь 2024"}],
    "sample_rows": [
        ["Клиент", "Город", "Сумма"],
        ["ООО Ромашка", "Москва", "100000"],
        ["ИП Иванов", "СПб", "50000"]
    ]
}
```

### Шаг 2: Формирование контекста
```python
context_text = """
Sheet: Отчет по продажам
Merged Cells: [{"top_left": "R1C1", "value": "Январь 2024"}]
Data Sample:
Row 1: ['Клиент', 'Город', 'Сумма']
Row 2: ['ООО Ромашка', 'Москва', '100000']
Row 3: ['ИП Иванов', 'СПб', '50000']
"""
```

### Шаг 3: Запрос к LLM (только если уровни 1-2 не справились)
```python
prompt = f"""
Ты — эксперт по анализу данных из 1С. Определи структуру файла.

{context_text}

Верни ответ в формате JSON:
{{
  "data_start_row": 3,
  "headers": [...],
  "hierarchy_levels": [...],
  "confidence_score": 0.87
}}
"""

# POST запрос к OpenRouter или Ollama
response = requests.post(
    "http://localhost:11434/api/generate",  # или OpenRouter URL
    json={"model": "llama3.2", "prompt": prompt, "format": "json"}
)
```

### Шаг 4: Получение результата
```json
{
  "data_start_row": 3,
  "headers": [
    {"original_name": "Клиент", "normalized_name": "customer"},
    {"original_name": "Сумма", "normalized_name": "revenue"}
  ],
  "hierarchy_levels": ["Группа товаров", "Товар"],
  "confidence_score": 0.87
}
```

## ПРЕИМУЩЕСТВА ТАКОГО ПОДХОДА

| Преимущество | Описание |
|-------------|----------|
| **Скорость** | 80-90% колонок обрабатываются за ~0.01 сек (без LLM) |
| **Экономия** | LLM вызывается только для 5-10% случаев |
| **Надёжность** | Если LLM недоступен, система работает на эвристиках |
| **Приватность** | Передаются только образцы, не весь файл |

## КАК НАСТРОИТЬ

### Вариант 1: OpenRouter (облачный, платный)
```bash
# 1. Получить API ключ на https://openrouter.ai/
export OPENROUTER_API_KEY=sk-or-v1-...

# 2. Запустить приложение
docker-compose up
```

**Стоимость:** ~$0.01-0.10 за запрос  
**Качество:** GPT-4, Claude, Llama-3-70B  

### Вариант 2: Ollama (локальный, бесплатный)
```bash
# 1. Установить Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 2. Скачать модель
ollama pull llama3.2

# 3. Настроить переменные окружения
export OLLAMA_HOST=http://localhost:11434
export OLLAMA_MODEL=llama3.2

# 4. Запустить приложение
docker-compose up
```

**Стоимость:** Бесплатно  
**Качество:** Llama-3.2 (хорошее)  
**Требования:** 8GB RAM, GPU желательно

## ВЫВОДЫ

1. **Система НЕ передаёт файлы целиком в LLM** — только текстовые образцы
2. **Трёхуровневая архитектура** экономит 70-80% токенов
3. **LLM — это последний рубеж**, а не основной метод
4. **Требуется настройка API ключей** для production
5. **Есть fallback на эвристики** если LLM недоступен
