# Расширенные нагрузочные тесты - Документация

## 📋 Обзор

Файл `tests/test_load.py` содержит расширенные сценарии нагрузочного тестирования для системы обработки документов. Тесты используют фреймворк **Locust** и покрывают 4 основных пользовательских сценария.

## 🎯 Пользовательские сценарии

### 1. DashboardUser - Типичный пользователь
**Описание:** Имитация работы обычного пользователя через веб-интерфейс

**Действия:**
- Просмотр главной страницы (вес: 5)
- Просмотр личного кабинета (вес: 4)
- Использование фильтров (быстрые пресеты + основные) (вес: 3)
- Расширенные фильтры по дате и сумме (вес: 2)
- Экспорт в PDF (вес: 2)
- Экспорт в PowerPoint (вес: 2)
- Просмотр деталей документа (вес: 1)

**Характеристики:**
- Задержка между действиями: 2-5 секунд
- Проверка status_code для всех запросов
- Реалистичные данные для фильтров

### 2. DataProcessor - Активная загрузка файлов
**Описание:** Стресс-тест для системы обработки документов

**Действия:**
- Загрузка Excel файлов (50-500 KB) (вес: 10)
- Загрузка PDF файлов с OCR (100-1000 KB) (вес: 5)
- Массовая загрузка (3-10 файлов подряд) (вес: 3)

**Характеристики:**
- Фиксированный интервал: 10 секунд между задачами
- Логирование медленных загрузок (>5 сек)
- Разные размеры файлов для реалистичности
- Увеличенные таймауты (30-60 сек)

### 3. ApiBot - API интеграции
**Описание:** Автоматизированная система для работы через API

**Действия:**
- Проверка статуса обработки (вес: 8)
- Поиск документов по различным критериям (вес: 6)
- Получение сводной аналитики (вес: 4)
- Запрос прогноза показателей (вес: 3)
- Массовые операции с документами (вес: 2)

**Характеристики:**
- Минимальная задержка: 0.5-2 секунды
- Быстрые запросы, имитирующие бота
- Разнообразные параметры поиска

### 4. StressTester - Предельная нагрузка
**Описание:** Экстремальные условия для проверки устойчивости

**Действия:**
- Серии быстрых запросов (rate limiting test) (вес: 20)
- Попытки одновременных загрузок (вес: 10)
- Сложные запросы с множеством фильтров (вес: 5)

**Характеристики:**
- Минимальный интервал: 0.1 секунды
- Ожидание ответов 200, 429, 503
- Генерация случайных поисковых запросов
- Тестирование всех типов фильтров одновременно

## 🚀 Запуск тестов

### Базовый тест (10 пользователей, 60 секунд)
```bash
locust -f tests/test_load.py --headless -u 10 -r 2 --run-time 60s --host http://localhost:8000
```

### Нагрузочный тест (100 пользователей, 5 минут)
```bash
locust -f tests/test_load.py --headless -u 100 -r 10 --run-time 300s --host http://localhost:8000
```

### Стресс-тест (500 пользователей, 10 минут)
```bash
locust -f tests/test_load.py --headless -u 500 -r 50 --run-time 600s --host http://localhost:8000
```

### Пиковая нагрузка (1000 пользователей)
```bash
locust -f tests/test_load.py --headless -u 1000 -r 100 --run-time 900s --host http://localhost:8000
```

### Веб-интерфейс для мониторинга
```bash
locust -f tests/test_load.py --web-port 8089 --host http://localhost:8000
```
Затем открыть в браузере: http://localhost:8089

## 📊 Параметры запуска

| Параметр | Описание | Пример |
|----------|----------|--------|
| `-u` | Общее количество пользователей | `-u 100` |
| `-r` | Скорость спавна (пользователей/сек) | `-r 10` |
| `--run-time` | Длительность теста | `--run-time 300s` |
| `--headless` | Режим без веб-интерфейса | `--headless` |
| `--host` | Целевой хост | `--host http://localhost:8000` |
| `--web-port` | Порт веб-интерфейса | `--web-port 8089` |

## 📈 Мониторинг метрик

### Выводимые метрики:
- **Всего запросов**: Общее количество выполненных запросов
- **Ошибок**: Количество неудачных запросов
- **Среднее время отклика**: Среднее время ответа сервера (ms)
- **Запросов в секунду (RPS)**: Текущая пропускная способность

### Логирование медленных запросов:
Автоматически логируются запросы с временем отклика > 3 секунд:
```
⚠️ SLOW REQUEST: GET /api/history - 4521ms
```

### Логирование медленных загрузок:
Для сценария DataProcessor логируются загрузки > 5 секунд:
```
⚠️ Slow upload detected: 6.23s for 500KB
```

## ✅ Критерии успеха

### Для базового теста (10-50 пользователей):
- ✅ Ошибки < 1%
- ✅ Среднее время отклика < 500ms
- ✅ 95-й перцентиль < 1000ms

### Для нагрузочного теста (100-200 пользователей):
- ✅ Ошибки < 2%
- ✅ Среднее время отклика < 1000ms
- ✅ 95-й перцентиль < 2000ms
- ✅ RPS стабилен

### Для стресс-теста (500+ пользователей):
- ✅ Ошибки < 5% (допускается rate limiting)
- ✅ Система не падает
- ✅ Восстановление после пика нагрузки

## 🔧 Настройка под свои нужды

### Изменение весов задач:
```python
@task(5)  # Увеличить вес для более частого выполнения
def my_task(self):
    ...
```

### Добавление новых эндпоинтов:
```python
@task(1)
def custom_endpoint(self):
    response = self.client.get("/api/custom")
    if response.status_code != 200:
        response.failure("Custom endpoint failed")
```

### Изменение времени ожидания:
```python
wait_time = between(1, 3)  # Случайная задержка 1-3 сек
wait_time = constant_pacing(5)  # Фиксированные 5 секунд
```

При возникновении проблем:
1. Проверить логи приложения
