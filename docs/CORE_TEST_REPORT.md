#### Уровень 1: Словарь синонимов (Dictionary)

- **Распознавание дат:** По формату значений (DD.MM.YYYY, YYYY-MM-DD)

#### Уровень 3: LLM (Ollama)

## ⚠️ Выявленные проблемы

### 1. Низкое распознавание сложных составных названий
**Проблема:** Колонки типа "КолвоШт", "ЦенаЗаЕдиницу", "КолПроданныхЕдиниц" не распознаются.

**Причина:** 
- Отсутствие полных синонимов в FIELD_MAPPINGS
- Эвристики не срабатывают на сложные составные слова

**Решение:**
```python
# Добавить в config/field_mappings.py:
"quantity": {
    "synonyms": [
        ...,
        "КолвоШт", "Колво", "КолПроданныхЕдиниц", 
        "КоличествоШтук", "КолЕд"
    ]
}
```

### 2. Иерархические данные
**Проблема:** Файл `hierarchical_data.xlsx` распознан только на 33%.

**Причина:**
- Колонка "Строка" с иерархическими значениями не маппится
- "КолПроданныхЕдиниц" — сложный синоним отсутствует

**Решение:**
- Добавить обработку иерархических колонок как отдельный тип
- Расширить словарь для количественных полей

### 3. Предупреждения FutureWarning
**Проблема:** Pandas предупреждает об устаревшем параметре `errors='ignore'`.

**Файлы:**
- `src/core/parser.py:218` — to_datetime
- `src/core/parser.py:228` — to_numeric

**Решение:** Обернуть в try-except вместо использования errors='ignore'
