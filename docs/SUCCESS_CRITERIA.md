# Критерии успеха проекта

## 📈 Метрики продукта (KPI)

### Активация
- **Цель:** 60% пользователей загружают первый файл в первые 24 часа
- **Измерение:** (Users who uploaded file / Total signups) × 100
- **Инструмент:** Amplitude / Mixpanel

### Удержание
- **Цель:** 40% пользователей активны через 30 дней
- **Измерение:** DAU/MAU ratio > 0.3
- **Инструмент:** PostgreSQL + Metabase

### Конверсия в оплату
- **Цель:** 10% trial пользователей переходят на платный тариф
- **Измерение:** (Paid users / Trial users) × 100
- **Инструмент:** Stripe / ЮKassa dashboard

### NPS
- **Цель:** NPS > 50
- **Измерение:** Опрос после 7 дней использования
- **Инструмент:** Typeform / Google Forms

### Прогнозирование (новые метрики)
- **Цель:** 70% пользователей смотрят прогнозы
- **Измерение:** (Users who viewed forecast / Total active users) × 100
- **Точность прогнозов:** MAPE < 15% для горизонтов до 3 месяцев
- **Инструмент:** Внутренняя аналитика + обратная связь

- [ ] 500 зарегистрированных пользователей
- [ ] 50 платящих клиентов

- [ ] 200 платящих клиентов

## 🔧 Технические метрики

### Производительность
- [ ] 99.9% uptime (SLA)
- [ ] API latency p95 < 500ms
- [ ] File processing time < 2 minutes (100K rows)
- [ ] Forecast building time < 5 seconds (100 points)
- [ ] Error rate < 1%

### Качество AI
- [ ] Column detection accuracy > 85%
- [ ] User confirmation rate < 30% (значит AI работает хорошо)
- [ ] Mapping learning: 50% файлов распознаются автоматически после 1 исправления

### Качество прогнозирования
- [ ] Auto-model selection accuracy > 80% (лучшая модель выбирается автоматически)
- [ ] MAPE < 15% для краткосрочных прогнозов (1-3 месяца)
- [ ] MAPE < 25% для долгосрочных прогнозов (6-12 месяцев)
- [ ] Доверительные интервалы калиброваны (90% фактических значений попадают в 95% ДИ)
- [ ] Поддержка 5+ методов прогнозирования
- [ ] Время подбора модели < 2 секунд

### Безопасность
- [ ] 0 критических уязвимостей
- [ ] Все данные шифруются at rest и in transit
- [ ] Compliance с 152-ФЗ
- [ ] Ежеквартальный security audit

## 📊 Метрики качества прогнозов

| Метрика | Формула | Цель | Критично |
|---------|---------|------|----------|
| **MAPE** | `avg(|actual - forecast| / actual)` | < 15% | Да |
| **MAE** | `avg(|actual - forecast|)` | Зависит от масштаба | Да |
| **RMSE** | `sqrt(avg((actual - forecast)²))` | Зависит от масштаба | Да |
| **MASE** | `MAE / MAE(naive forecast)` | < 1.0 | Да |
| **Coverage** | `% actuals within confidence interval` | 90-95% | Да |
| **Bias** | `avg(forecast - actual)` | Близко к 0 | Нет |

### Бенчмарки по методам
| Метод | Ожидаемый MAPE | Требования к данным |
|-------|---------------|---------------------|
| Exponential Smoothing | 10-20% | 20+ точек |
| ARIMA | 8-15% | 50+ точек, стационарность |
| Prophet | 5-12% | 100+ точек, сезонность |
| XGBoost | 5-10% | 100+ точек, много признаков |
| Ensemble | 3-8% | 200+ точек |
