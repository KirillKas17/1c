# Реализация расширенных фильтров веб-интерфейса

## Обзор

Реализована полноценная система фильтрации и сортировки для вкладки "История обработки документов".

## Реализованные функции

### 1. Быстрые пресеты (Quick Presets)
Кнопки для быстрого применения часто используемых фильтров:
- **Сегодня** - документы, обработанные сегодня
- **Неделя** - документы за последние 7 дней
- **Месяц** - документы за последние 30 дней
- **Только ошибки** - документы со статусом "Ошибка"
- **Сбросить всё** - очистка всех фильтров

### 2. Основные фильтры
- **Поиск** - полнотекстовый поиск по имени файла, ИНН, контрагенту, номеру документа
- **Тип документа** - выбор типа (счёт, акт, УПД, накладная и т.д.)
- **Статус** - успешно/частично/ошибка
- **Контрагент** - выбор из списка всех контрагентов
- **Сортировка** - 8 вариантов сортировки:
  - По дате обработки (новые/старые)
  - По дате документа (новые/старые)
  - По сумме (возрастание/убывание)
  - По имени файла (А-Я/Я-А)

### 3. Расширенные фильтры
- **Дата с/по** - диапазон дат документа
- **Сумма от/до** - диапазон сумм документа

## Технические детали

### Frontend (web/index.html)

#### Новые UI компоненты:
```html

<button onclick="applyPreset('week')">Неделя</button>

  <option value="processed_at_desc">По дате обработки (новые)</option>

  <option value="document_date_asc">По дате документа (старые)</option>
```

#### JavaScript функции:
```javascript
// Применение пресета
applyPreset(preset) - устанавливает даты или статус

// Сброс фильтров
clearFilters() - очищает все поля и перезагружает данные

// Загрузка истории с фильтрами
loadHistory() - собирает все фильтры и отправляет на backend
```

### Backend (core_parser/utils/processing_history.py)

#### Новые параметры фильтрации:
```python
filters = {
    'search': str,           # Поиск по тексту
    'document_type': str,    # Тип документа
    'status': str,           # Статус
    'contractor_inn': str,   # ИНН контрагента
    'date_from': str,        # Дата документа с (YYYY-MM-DD)
    'date_to': str,          # Дата документа по (YYYY-MM-DD)
    'amount_from': float,    # Сумма от
    'amount_to': float,      # Сумма до
    'sort_by': str           # Сортировка
}
```

#### SQL запросы:
- Динамическое построение WHERE условий
- Поддержка ORDER BY с маппингом параметров
- Фильтрация по числовым диапазонам (суммы)

## Использование

### Через UI:
1. Откройте вкладку "История"
2. Используйте быстрые пресеты или настройте фильтры вручную
3. Нажмите "Применить фильтры" или используйте авто-обновление

### Через API:
```python
from core_parser.utils.processing_history import ProcessingHistory

history = ProcessingHistory()

# Получить документы за неделю с суммой от 10000
result = history.get_history(
    page=1,
    per_page=50,
    filters={
        'date_from': '2024-01-01',
        'date_to': '2024-01-07',
        'amount_from': 10000.0,
        'sort_by': 'amount_desc'
    }
)
```

## Адаптивность

Интерфейс фильтров полностью адаптивен:
- **Desktop (XL)**: 6 колонок для фильтров
- **Laptop (LG)**: 4 колонки для расширенных фильтров
- **Tablet (MD)**: 2 колонки
- **Mobile**: 1 колонка, вертикальное расположение

## Производительность

- Debounce поиска: 500ms
- Пагинация на уровне БД
- Индексы SQLite для полей фильтрации
