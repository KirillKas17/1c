# 🔍 LLM ARCHITECTURE DOCUMENTATION

## Как система использует LLM для анализа файлов

### 📋 КРАТКИЙ ОТВЕТ НА ВАШ ВОПРОС:

**Мы НЕ передаём файл целиком в LLM.** 

Система использует **гибридный трёхуровневый подход**:

```
┌─────────────────────────────────────────────────────────┐
│  УРОВЕНЬ 1: Словарь синонимов (быстро, без LLM)         │
│  → Сопоставление названий колонок с известыми полями 1С │
│  → Confidence: 0.95-0.98                                │
└─────────────────────────────────────────────────────────┘
                          ↓ (если confidence < 0.9)
┌─────────────────────────────────────────────────────────┐
│  УРОВЕНЬ 2: Эвристики (быстро, без LLM)                 │
│  → Анализ форматов данных (дата, число, строка)         │
│  → Паттерны названий, иерархии                          │
│  → Confidence: 0.75-0.90                                │
└─────────────────────────────────────────────────────────┘
                          ↓ (если confidence < 0.6)
┌─────────────────────────────────────────────────────────┐
│  УРОВЕНЬ 3: LLM (Ollama/OpenRouter) - ТОЛЬКО ДЛЯ        │
│           сложных случаев                               │
│  → Передаётся ТЕКСТОВЫЙ контекст, а не файл            │
│  → Confidence: 0.5-0.8                                  │
└─────────────────────────────────────────────────────────┘
```

## 🏗️ ДЕТАЛЬНАЯ АРХИТЕКТУРА

### 1. Компоненты системы

#### A. `AIDetector` (src/core/ai_detector.py)
**Задача:** Распознавание отдельных колонок на уровне полей

**Как работает:**
```python
detector = AIDetector(ollama_url="http://localhost:11434", model="llama3.2")

# Для каждой колонки:
mappings = detector.detect_columns(
    column_names=["Клиент", "СуммаПродажи", "ДатаДокумента"],
    sample_data={"Клиент": ["ООО Ромашка", "ИП Иванов"], ...}
)
```

**Что передаётся в LLM (только если уровни 1-2 не справились):**
```python
# Формируется текстовый промпт:
prompt = f"""
Ты - эксперт по анализу данных из 1С. Определи, что означает колонка 
с названием "{col_name}".

Примеры значений: {sample_str}  # Первые 5 значений

Возможные типы полей в 1С:
- revenue (Выручка, СуммаПродажи, Оборот)
- cost (Себестоимость, Затраты)
- quantity (Количество, Вес, Объём)
- customer (Клиент, Контрагент, Покупатель)
...

Верни ответ в формате JSON:
{{
  "mapped_field": "тип_поля",
  "confidence": 0.0-1.0,
  "data_type": "string|int|float|datetime",
  "reasoning": "краткое обоснование"
}}
"""
```

**Важно:** 
- ❌ НЕ передаётся весь файл
- ❌ НЕ передаются бинарные данные
- ✅ Передаётся только: название колонки + 5 примеров значений

#### B. `AIExcelAnalyzer` (src/ai_core/ai_analyzer.py)
**Задача:** Глобальный анализ структуры файла (иерархия, объединённые ячейки, фильтры)

**Как работает:**
```python
analyzer = AIExcelAnalyzer()
result = analyzer.analyze_file("path/to/report.xlsx")
```

**Этапы работы:**

##### Этап 1: Извлечение образца (БЕЗ LLM)
```python
sample = analyzer.extract_sheet_sample(file_path, max_rows=15)

# Возвращает:
{
  "sheet_name": "Отчет по продажам",
  "merged_cells": [
    {"top_left": "R1C1", "size": "1x3", "value": "Январь 2024"}
  ],
  "sample_rows": [
    ["Клиент", "Город", "Сумма", "Вес"],
    ["ООО Ромашка", "Москва", "100000", "50"],
    ["ИП Иванов", "СПб", "50000", "25"],
    ...
  ]
}
```

##### Этап 2: Формирование контекста для LLM
```python
context_text = """
Sheet: Отчет по продажам
Merged Cells Info: [{"top_left": "R1C1", "size": "1x3", "value": "Январь 2024"}]
Data Sample (Row by Row):

Row 3: ['ИП Иванов', 'СПб', '50000', '25']
...
"""
```

##### Этап 3: Запрос к LLM (OpenRouter или Ollama)
```python
# СИСТЕМНЫЙ ПРОМПТ (жёстко задан в коде):
system_prompt = """
```

## 🔧 КОНФИГУРАЦИЯ LLM

### Переменные окружения

```bash
# .env.production (необходимо добавить!)

# OpenRouter (основная модель)
OPENROUTER_API_KEY=your-api-key-here
OPENROUTER_MODEL=meta-llama/llama-3-70b-instruct

# Ollama (локальный fallback)
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=llama3.2

# Пороговые значения confidence
LLM_CONFIDENCE_THRESHOLD_DICTIONARY=0.9
LLM_CONFIDENCE_THRESHOLD_HEURISTIC=0.7
LLM_CONFIDENCE_THRESHOLD_LLM=0.6
```

### Текущее состояние

⚠️ **ПРОБЛЕМА:** В проекте НЕТ настроенного `.env` файла с API ключами!

**Что есть:**
- ✅ Код поддерживает OpenRouter (через `OPENROUTER_API_KEY`)
- ✅ Код поддерживает локальную Ollama (по умолчанию `http://localhost:11434`)
- ❌ Нет `.env` файла с реальными ключами
- ❌ Нет `.env.example` с секцией AI/LLM настроек

## 🚀 ПРЕИМУЩЕСТВА ТАКОГО ПОДХОДА

| Преимущество | Описание |
|-------------|----------|
| **Скорость** | 80-90% колонок обрабатываются без LLM (уровни 1-2) |
| **Экономия** | LLM вызывается только для сложных случаев (~10-20%) |
| **Надёжность** | Если LLM недоступен, система работает на эвристиках |
| **Гибкость** | Можно переключаться между OpenRouter и локальной Ollama |
| **Приватность** | Передаются только образцы данных, не весь файл |

## ⚠️ ПРОБЛЕМЫ И РИСКИ

### 1. Отсутствие API ключей
**Статус:** КРИТИЧНО  
**Решение:** Добавить в `.env`:
```bash
OPENROUTER_API_KEY=sk-or-...
```

### 2. Нет документации по настройке Ollama
**Статус:** СРЕДНИЙ  
**Решение:** Создать `docs/OLLAMA_SETUP.md`

### 3. LLM может быть недоступен в production
**Статус:** СРЕДНИЙ  
**Решение:** Уже реализован fallback на эвристики

### 4. Высокая задержка при вызове LLM
**Статус:** НИЗКИЙ  
**Решение:** Кэширование результатов (`_llm_cache`)

## 🔐 БЕЗОПАСНОСТЬ

### Что передаётся в LLM:
✅ Названия колонок (текст)  
✅ Первые 5-10 значений (текст)  
✅ Метаданные о merged cells (текст)  
✅ Системный промпт (жёстко задан)  

### Что НЕ передаётся:
❌ Весь файл целиком  
❌ Бинарные данные  
❌ Конфиденциальные данные клиентов (передаются только образцы)  
❌ Пароли, API ключи, токены  

### Рекомендации:
1. Использовать локальную Ollama для максимальной приватности
2. Не передавать реальные персональные данные в OpenRouter
3. Кэшировать результаты для повторных структур

## 🛠️ КАК НАСТРОИТЬ

### Вариант 1: OpenRouter (облачный, платный)
```bash
# 1. Получить API ключ на https://openrouter.ai/
export OPENROUTER_API_KEY=sk-or-v1-...

# 2. Запустить приложение
docker-compose up
```

**Плюсы:**
- ✅ Не нужно устанавливать модели локально
- ✅ Высокое качество (GPT-4, Claude, Llama-3-70B)
- ✅ Быстрый старт

**Минусы:**
- ❌ Платно (~$0.01-0.10 за запрос)
- ❌ Данные уходят во внешнюю службу

---

### Вариант 2: Ollama (локальный, бесплатный)
```bash
# 1. Установить Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 2. Скачать модель
ollama pull llama3.2

# 3. Настроить переменные окружения
export OLLAMA_HOST=http://localhost:11434
export OLLAMA_MODEL=llama3.2

# 4. Запустить приложение
docker-compose up
```

**Плюсы:**
- ✅ Бесплатно
- ✅ Полная приватность (данные не покидают сервер)
- ✅ Работает офлайн

**Минусы:**
- ❌ Требует ресурсов (4-8 GB RAM, GPU желательно)
- ❌ Нужно устанавливать и поддерживать модели
- ❌ Качество может быть ниже чем у GPT-4

## 🎯 ВЫВОДЫ

1. **Система НЕ передаёт файлы целиком в LLM** — только текстовые образцы
